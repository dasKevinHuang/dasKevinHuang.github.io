Writing Code #4: Enumerable#map December 5, 2015

When you are just starting out learning how to program (like I am) you will encounter arrays and hashes. These two types of collections of objects are especially important in learning how to write competent code, so pay attention to how they work! Let's review the properties of arrays and hashes.

What is map? Why bother using it?

map is a method that returns a new array based on the running code block pass into #map. map is important because it makes it easy to create a new array from an object depending on what you would like to do with it.

To better help you understand how #map works, you could compare it to something at it definitely is not: each. each on the other hand, does NOT return a new array and will simply iterate over each element of an array using the running block. So if you are writing code using each and expect it to return a new array, it simply will not do it - unless you modify the original array.

This is where map comes in handy! Since map returns a new array based on the running block, you can be assured that a newly modified array based on your requirements will greet you after running it!

To visualize this difference here is an example of each method: 

        
        a = [5, 6, 7, 8]
        a.each {|x| x - 5}
        
        // This will return the original array, unmodified - essentially doing nothing.
        // [5, 6, 7, 9]

        a = [5, 6, 7, 8]
        a.map {|x| x - 5}

        // This will return a NEW array
        // [0, 1, 2, 3]
