Writing Code #3: Arrays and Hashes December 5, 2015

When you are just starting out learning how to program (like I am) you will encounter arrays and hashes. These two types of collections of objects are especially important in learning how to write competent code, so pay attention to how they work! Let's review the properties of arrays and hashes.

What's an array again? 

        
        //Here are some examples of arrays in code.

        //Declares an empty array
        
        array_1 = []

        //Defines an array with 2, 3, 4, 5 inside.
        
        array_2 = [2, 3, 4, 5]

        //Using an index to access element 3 of array_2
        //Note how index numbers start from 0, not 1.
        
        puts array_2[2]

        //This outputs 4
Arrays are structured as a collection of elements, with each element being identified by an index number. Array indices begin with the number 0.

Array elements contain the actual values that are inside the array. Array elements can be manipulated with code. You can also use an index to point to a specific element in an array.

What's a hash then? 

        
        //Here are some examples of hashes in code.

        //Declares an empty hash
        
        blah = {}

        //Defines key-value pairs for the hash

        blah[name] = "Not My Real Name"
        blah[power_level] = "Over 9000"
A hash is data structure that is sort of like an array, however instead of relying on index numbers to point to specific elements, hashes use hash keys. Hash elements actually come in 2 parts called key-value pairs. The hash key is the part that identifies a specific hash value, and the hash value of course is the value itself.

What's so special about a hash?

It's important to know that with hash keys, order isn't as importantas when dealing with arrays. While in arrays order is important, with hash keys, it's the hash pairs that are important. When dealing with hashes, you have to be careful with accessing the a value with the correct key.

Another interesting aspect to hash keys that is different from arrays is that hash keys must be unique. This is especially important since order doesn't matter as much as with arrays when dealing with hash. So the hash key you assign must be unique.

What's the difference between an array and a hash?

The main differences between arrays and hashes are that for:
Arrays
+ Order is very important.
+ Allows for copies of the same thing.

Hashes:
+ Hash keys must be unique.
+ No duplicate hash keys are allowed.

To illustrate this, take a look at what arrays can do: 

        
        a = [1, 1, 1, 2]
Do you see how arrays can hold multiple copies of the same thing? Now let's take a look at when we try this with a hash: 
        
        blah = {}
        blah[:a] = "A"
        blah[:a] = "B"
Did you catch what happens with this code?
The 3rd line actually overwrites the previous value of this key-value pair, since hash keys are unique.
